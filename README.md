# Visual With

## 설계

- 팀 초대받는 화면 설계

## 뭘 해야할까

- 화면의 명확한 세션 구분
- 모달 구현
- 네비게이션의 css 구조 조정
- 화면 내 세부 요소 구현
- 라우팅: 라우팅 배운 후 진행하면 됨
- 전역 ui 폴더 만들기 (모달 구현 시 우선순위 조정 가능)
- 전체 화면 디자인
- 버튼 아이콘 등록
- 세부 요소 디자인
- 요소별 UI 반응 구현 (호버링 등)

## 완료된 것

- 핵심 화면: 친구 화면, 친구 채팅 화면
- 도구 간 이동(버튼 동작)
- 메인 페이지의 각 페이지별 헤더 전달: 과연 이게 최선일까? -> 메인바디를 중간자로 두어서 해결
- 헤더, 푸터 css 구조 조정

## 미래에

- src(루트)에 공용 hooks/, utils/, 각 features에 전용 hooks/, utils/를 두어 관리
- store/도 루트에 필요하다면 두고 관리
